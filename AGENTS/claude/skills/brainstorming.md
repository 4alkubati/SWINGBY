# Skill: brainstorming

> Borrowed from Superpowers. The guardrail against Kira's "leave the spec blank" pattern.

## When it triggers
Before any new feature or build task — *before* code, before filling 5W+H.

## The move
Don't jump to building. Step back and ask what's *really* being built:
1. Ask sharp questions until the goal is concrete (one at a time — Kira's rule).
2. Explore 2–3 alternatives, not just the first idea.
3. Present the spec back **in small chunks short enough to actually read.**
4. Get explicit sign-off on each chunk before moving on.
5. Save the agreed spec → it becomes the task's 5W+H.

## Why this fits Kira
His failure pattern is fleeing the moment he'd have to commit to an answer that might be wrong (the blank SwingBy template). Brainstorming forces the commitment into small, low-stakes pieces — so the scary "what is this exactly?" gets answered before it can trigger a flee.
