# _SKILLS — methodology playbooks

Back to [[MAP]]. Plain-English procedures every agent follows. Borrowed from Superpowers + Hermes, adapted to Kira's system.

## Process (the spine of the gate)
- [[brainstorming]] — tease the spec before code (Layer 1)
- [[writing-plans]] — 2–5 min tasks, exact paths + code + checks (Layer 2–3)
- [[two-stage-review]] — spec compliance, then code quality (Layer 6)
- [[verification-before-completion]] — "done" = proven, not hoped (Layer 6)

## Power tools
- [[systematic-debugging]] — 4-phase root cause (kills the can't-debug bottleneck)
- [[learning-loop]] — the system improves itself every cycle (Layer 7)

---
*All skills are mandatory, not optional. The [[ORCHESTRATOR]] checks for a relevant skill before every task.*

## Quality (from the 6-skills method)
- [[ingest-source]] — capture a source into compacted, role-sliced knowledge (Layer 1)
- [[ask-the-board]] — expert panel (engineer/security/designer/PM) reviews high-stakes work (Layer 6)
- [[internal-focus-group]] — simulate real users on a screen/flow before ship (Layer 6)
- [[impeccable-design]] — pro-grade UI; kills "AI-built" tells; mandatory for any screen work (Layer 6)
