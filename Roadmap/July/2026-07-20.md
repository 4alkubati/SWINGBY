## 📖 LOG — overnight session (Jul 19 night → Jul 20 early)

### ✅ First successful Android build. Ever, with maps wired.
`eas build --platform android --profile preview` **succeeded**. Getting there took clearing four separate defects that were stacked on the same command, each one hiding the next:

1. **Sentry killed Gradle** — `@sentry/react-native` is enabled as a plugin with no org configured, so the release build's source-map upload had nothing to authenticate with (`sentry.gradle:149`). EAS builds run in the cloud, so a local env var could never fix it — declared `SENTRY_DISABLE_AUTO_UPLOAD=true` per-profile in `eas.json`, plus `buildType: apk` so the artifact is sideloadable.
2. **`app.config.js` was ESM in a CommonJS package** — `export default` with no `"type": "module"` is a hard SyntaxError, so `expo config --json` exited 1 and took down *every* expo/eas command, including `env:create`.
3. **`expo-local-authentication` wasn't installed** — the branch collapse added it as a dependency and plugin; `node_modules` predated the merge, so plugin resolution failed with the same opaque exit-1.
4. **Wrong Android Maps schema** — Android needs `android.config.googleMaps.apiKey` (nested); both `app.json` and `app.config.js` used the flat iOS shape `googleMapsApiKey`. Only `expo doctor` in the cloud catches this; nothing local validates the schema.

**All four came from `950d4f4` (CARD-03) — a commit whose agent exited before committing or reporting, so the file had never once been executed.** The Google Maps key could never have reached any build, whatever was pasted where. This is the done-rule's whole point.

## 🎓 LEARNING
- **A cloud build validates things local commands never do.** `expo doctor` runs in EAS and not on your laptop, so schema errors survive indefinitely in a repo where nobody builds.
- **`EXPO_PUBLIC_` variables cannot be secret** — Expo rejects `--visibility secret` for them because they compile into the bundle. Restriction in Google Cloud is the only protection, not secrecy.
- **After any pull touching `mobile/`, run `npm install` before any expo command.** A declared-but-missing plugin fails with an opaque exit-1 that never names the package.
- **"Committed and pushed" is not done.** Four defects shipped inside one commit that was never executed.
