# AUGUST — Phase 3: Public Launch

> The finish line. SwingBy goes public and real users transact. Deadline: **Aug 31.**

## August win condition
By Aug 31: **SwingBy is live to the public** (web first, stores as they clear review), real businesses + clients signed up, real bookings happening with live payments. The Aug 31 goal = met.

## Weekly themes

| Week | Theme | Key moves |
|---|---|---|
| **W1 (Aug 1–7)** | Soft public launch | Open the web app publicly; convert waitlist → users; monitor closely |
| **W2 (Aug 8–14)** | Stores go live | App Store + Play approvals land → public; fix launch-day bugs fast |
| **W3 (Aug 15–21)** | Drive signups | Push outreach/PR (FOH agents); local Calgary trades + clients; referral loop |
| **W4 (Aug 22–31)** | Scale + harden | Upgrade Supabase/Render if load needs it; watch metrics; close the goal |

## Launch-day checklist (W1/W2)
- Backend always-on (Render paid, not free/spin-down).
- Stripe **live** keys in, escrow verified.
- Emails firing (Resend verified domain).
- Store listings approved + privacy disclosures matching.
- Monitoring on (Sentry alerts → Telegram).

## New automations
- **gen-4:** launch-day broadcast (announce to waitlist across channels).
- **gen-5:** daily growth report (signups, bookings, revenue) → Telegram.

## The bar
Aug 31, you can say: *"SwingBy is live, people use it for their business, and I proved I could ship it."* That's the win you named on day one.
