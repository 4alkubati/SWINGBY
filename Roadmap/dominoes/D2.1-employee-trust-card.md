## 🎯 Goal

[[../../mobile/src/screens/business/EmployeeProfileScreen|EmployeeProfileScreen]] shows the same caliber of trust signals as [[../../mobile/src/screens/business/BusinessProfileScreen|BusinessProfileScreen]]: real avg rating + count, "Verified by {Business}" badge, "{N} jobs completed", "Member since {Mon YYYY}".

## 🪜 Step-by-step

### 1. Backend — add aggregation endpoint
File: `[[../../backend/app/api/employees.py]]` (new or extend existing).

Endpoint:
```
GET /employees/{employee_id}/profile
→ {
  id, user_id, business_id, role_title, is_active, joined_at,
  avg_rating: float | null,
  review_count: int,
  jobs_completed: int,
  verified_via_business: bool,        # mirrors businesses.license_status == 'verified'
  business_name: str
}
```

Implementation notes:
- avg_rating + count: `SELECT avg(rating), count(*) FROM reviews WHERE reviewee_id = :uid AND reviewee_type = 'employee'`.
- jobs_completed: `SELECT count(*) FROM bookings WHERE employee_id = :eid AND status = 'completed'`.
- verified_via_business: join `businesses.license_status`.
- joined_at: `employees.created_at`.
- Auth: any authenticated user can read.

**Verify:** `curl -H "Authorization: Bearer ..." https://swingbyy-api.onrender.com/employees/<id>/profile` returns the shape.

### 2. Backend — register route in [[../../backend/app/main.py|main.py]]
Add `app.include_router(employees.router, prefix="/employees", tags=["employees"])` if not already there.
**Verify:** `/openapi.json` shows the route.

### 3. Mobile — extend [[../../mobile/src/screens/business/EmployeeProfileScreen|EmployeeProfileScreen]]
- Swap the placeholder fetch for `GET /employees/{id}/profile`.
- Render in the existing identity card:
  - Stars (use existing `RatingStarsDisplay`) + count next to it ("4.8 · 23 reviews").
  - If `verified_via_business`: render a `Badge` "Verified by {business_name}" (success color).
  - Stats row → 3 boxes: rating · jobs_completed · "Since {Mon YYYY}".
- Loading state: skeleton already exists, just align the new fields.
- Empty state: if `review_count === 0`, show "No reviews yet — first one earns a badge."

### 4. Mobile — link from [[../../mobile/src/screens/business/EmployeeManagementScreen|EmployeeManagementScreen]]
Already navigates to EmployeeProfile — sanity check the navigation params include `employeeId`.

### 5. Manual test

- **Verify:** trust card renders all 4 fields. No "—" placeholders. Verified badge shown only when business is verified.
