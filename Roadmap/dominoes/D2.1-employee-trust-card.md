## ЁЯкЬ Step-by-step

### 1. Backend тАФ add aggregation endpoint
File: `[[../../backend/app/api/employees.py]]` (new or extend existing).

Endpoint:
```
GET /employees/{employee_id}/profile
тЖТ {
  id, user_id, business_id, role_title, is_active, joined_at,
  avg_rating: float | null,
  review_count: int,
  jobs_completed: int,
  verified_via_business: bool,        # mirrors businesses.license_status == 'verified'
  business_name: str
}
```

Implementation notes:
- avg_rating + count: `SELECT avg(rating), count(*) FROM reviews WHERE reviewee_id = :uid AND reviewee_type = 'employee'`.
- jobs_completed: `SELECT count(*) FROM bookings WHERE employee_id = :eid AND status = 'completed'`.
- verified_via_business: join `businesses.license_status`.
- joined_at: `employees.created_at`.
- Auth: any authenticated user can read.

**Verify:** `curl -H "Authorization: Bearer ..." https://swingbyy-api.onrender.com/employees/<id>/profile` returns the shape.
