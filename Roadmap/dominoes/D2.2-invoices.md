# 🁢 D2.2 — Invoices (in-app receipt + downloadable PDF)

> Index: [[../DOMINOES|DOMINOES]] · Prev: [[D2.1-employee-trust-card|D2.1]] · Next: [[D2.3-offplatform-pay|D2.3 — Off-platform pay]] · Master log: [[_LEARNING-LOG]]

## 🎯 Goal

After a completed booking, both client and business can:
1. Open an in-app **Receipt** screen showing line items, totals, platform cut, payment method, booking ID, parties.
2. Tap **Download PDF** to open a real PDF in iOS Safari that they can save / share / email.

## 🤔 Why this matters

Real businesses need real paperwork. A receipt screen + downloadable PDF is the difference between "a hobby app" and "I can use this for my actual books." It's also the artifact a dispute hangs on.

## 🪜 Step-by-step

### 1. Backend — pip add a PDF library
File: `[[../../backend/requirements.txt]]`
Add: `reportlab>=4.0` (pure Python, no system deps — works on Render).
**Verify:** `pip install -r requirements.txt` clean locally.

### 2. Backend — invoice data endpoint
File: `[[../../backend/app/api/invoices.py]]` (new).

```
GET /bookings/{booking_id}/invoice
→ JSON shape:
{
  invoice_number: "SWB-{YYYY}-{0000N}",   # see below
  issued_at, booking_id,
  client: {name, email},
  business: {name, address, license_status},
  employee: {name, role_title} | null,
  service: {title, description, category},
  schedule: {requested_date, completed_at},
  line_items: [
    {label: "Service", amount: 200.00},
    {label: "Platform fee (10%)", amount: -20.00},
  ],
  totals: {subtotal: 200.00, platform_cut: 20.00, paid_to_business: 180.00, total_charged: 200.00},
  payment: {method: "stripe_card" | "cash" | "e_transfer" | "other", status: "paid_full" | "paid_off_platform", processor_ref: "ch_..." | null}
}
```

Notes:
- `invoice_number`: generate from `bookings.created_at` year + a row count. Easiest: store `invoices.id` or just use the booking ID prefixed `SWB-`.
- Auth-gate: only `bookings.client_id` or `bookings.business.owner_id` can read.

**Verify:** `curl ... /bookings/<id>/invoice` returns the JSON.

### 3. Backend — PDF endpoint
Same file, sibling route:
```
GET /bookings/{booking_id}/invoice.pdf
→ application/pdf, inline or attachment.
```
- Render the JSON above through `reportlab.platypus` (SimpleDocTemplate + Table + Paragraph).
- Stream the bytes back via `StreamingResponse` from `io.BytesIO`.
- Header: `Content-Disposition: inline; filename="SwingBy-Invoice-{id}.pdf"`.
**Verify:** open the URL in a browser → PDF renders with all fields.

### 4. Backend — register router
`[[../../backend/app/main.py]]` → `app.include_router(invoices.router, tags=["invoices"])`.

### 5. Mobile — new InvoiceScreen
File: `[[../../mobile/src/screens/shared/InvoiceScreen.js]]` (shared bucket — both roles use it).

- Fetches `/bookings/{id}/invoice` on mount.
- Renders sections: header (logo + invoice_number) → parties → service → schedule → line items table → totals → payment row → footer.
- "Download PDF" button → `Linking.openURL` with the PDF URL + auth token via query param OR (better) a signed short-lived URL endpoint.
- Loading + empty + error states (per [[../../AGENTS/claude/PRODUCT-VISION|PRODUCT-VISION]] mobile-agent rule).

### 6. Mobile — wire navigation
- Add `<Stack.Screen name="Invoice" component={InvoiceScreen} />` to both [[../../mobile/src/navigation/ClientNavigator.js|ClientNavigator]] + [[../../mobile/src/navigation/BusinessNavigator.js|BusinessNavigator]].
- In [[../../mobile/src/screens/client/BookingDetailsScreen|BookingDetailsScreen]] and [[../../mobile/src/screens/business/JobManagementScreen|JobManagementScreen]]: when `booking.status === 'completed'`, show a primary "View receipt" button that pushes Invoice.

### 7. PDF download auth
Two options — pick one:
- **A — query token (simpler, less secure):** mobile appends `?token=<jwt>` to the URL; backend reads + verifies.
- **B — signed download (better):** new endpoint `POST /bookings/{id}/invoice/signed-url` returns a 5-minute signed URL; mobile opens that.
Beta = **A** is fine. Note this as tech debt in [[_LEARNING-LOG]].

- Complete a sandbox booking end-to-end on phone.
- Tap "View receipt" → in-app screen renders.
- Tap "Download PDF" → iOS Safari opens, PDF visible, share sheet works.
