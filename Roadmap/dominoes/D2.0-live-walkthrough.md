# 🁢 D2.0 — Live walk-through audit

> Index: [[../DOMINOES|DOMINOES]] · Next: [[D2.1-employee-trust-card|D2.1 — Employee trust card]] · Master log: [[_LEARNING-LOG]]

## 🎯 Goal

Tap through every reachable screen of the SwingBy mobile app on **Expo Go iOS pointed at the LIVE Render backend**, and log every breakage to today's daily file.

## 🤔 Why this comes first

Before we add invoices, subscriptions, employee ratings, off-platform pay — we need to know what's already broken. Otherwise we're stacking new features on top of latent bugs. This domino is the cheap insurance pass.

## ✅ Pre-reqs

- [ ] Mobile `[[../../mobile/.env]]` already has `EXPO_PUBLIC_API_URL=https://swingbyy-api.onrender.com` (verified ✅).
- [ ] `[[../../AGENTS/claude/memory/STATUS|STATUS.md]]` reviewed for known issues so you don't relog them.
- [ ] iPhone with Expo Go installed (free, App Store).
- [ ] Render dyno warm (curl `https://swingbyy-api.onrender.com/healthz` — first request after idle is ~22 s cold start).

## 🪜 Step-by-step

### 1. Warm Render and start Expo
```bash
curl https://swingbyy-api.onrender.com/healthz
cd mobile
npx expo start --clear
```
**Verify:** terminal shows a QR code and metro bundler is ready.

### 2. Open Expo Go on iPhone and scan QR
**Verify:** app loads to [[../../mobile/src/screens/onboarding/OnboardingScreen|OnboardingScreen]] within 30 s.

### 3. Walk the AUTH bucket
Screens to touch: [[../../mobile/src/screens/auth/LoginScreen|Login]] · [[../../mobile/src/screens/auth/SignupScreen|Signup]] · [[../../mobile/src/screens/auth/ForgotPasswordScreen|ForgotPassword]].
- Sign up as a brand-new throwaway client account → confirm branded email lands in your inbox.
- Try forgot-password → confirm reset email lands.
- Log out, log in with existing test client (`testclient@swingby.dev / SwingBy2024!`).
**Verify:** lands on [[../../mobile/src/screens/client/HomeScreen|HomeScreen]].

### 4. Walk the CLIENT bucket
Order: Home → NearbyMap → Search → PostJob → MyJobs → BookingDetails → QuoteComparison → ActiveBooking → Review → Favorites → Map → Notifications → NotificationsCenter → PaymentMethod → Referral → HelpFAQ → PrivacyPolicy → ProfileEdit → Settings.
- For each: scroll once, tap any obvious action, screenshot any breakage.
- Note loading states + empty states (per [[../../AGENTS/claude/PRODUCT-VISION|PRODUCT-VISION]]: "Every list/detail screen has loading/empty/error states").

### 5. Walk the BUSINESS bucket (log out, log in as `testbusiness@swingby.dev`)
Order: Dashboard → MyBusiness (BusinessProfile) → Jobs (MyJobs) → JobManagement (Status / Details / Messages tabs) → Earnings → BusinessAnalytics → EmployeeManagement → EmployeeProfile → BookingDetails.
- Pay close attention to [[../../mobile/src/screens/business/EmployeeProfileScreen|EmployeeProfileScreen]] — we already know it's a [[D2.1-employee-trust-card|D2.1]] gap (no trust signals, ratings show "—").

### 6. Walk the SHARED + flows + messages buckets
[[../../mobile/src/screens/messages/MessagesScreen|Messages]] → [[../../mobile/src/screens/messages/ChatScreen|Chat]] → [[../../mobile/src/screens/flows/CancellationFlowScreen|Cancellation]] → [[../../mobile/src/screens/flows/DisputeFlowScreen|Dispute]].
- Send one test message in an existing thread → confirm it round-trips.

### 7. File the bug list
Open [[../June|June.md]] and append a section under the Jun 28 entry:

```markdown
## D2.0 — Walk-through bugs
- [ ] Screen: <name> · severity: blocker/major/minor · what: ... · expected: ... · actual: ...
```

### 8. Triage the list
Mark each bug:
- 🔴 **blocker** — beta cannot proceed (e.g., crash, infinite spinner, auth broken)
- 🟡 **major** — bad UX but flow completable
- 🟢 **minor** — cosmetic / polish

Blockers fold back into the active sub-domino. Majors go to [[D3-expo-go-walkthrough|D3]]. Minors get parked.
